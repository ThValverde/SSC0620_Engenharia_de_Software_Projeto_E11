🏗️ ARQUITETURA RBAC (4 Níveis)
═══════════════════════════════════════════════════════════════════════════════

Level 4: Superuser (Admin Django)
  • is_superuser: true
  • Pode criar: Secretaria_Admin
  • Acesso: TODOS os módulos

Level 3: Secretaria_Admin (OTO Admin)
  • groups: ['Secretaria_Admin']
  • Pode criar: Secretaria_Staff, Trade
  • Acesso: Todos exceto nivel super admin

Level 2: Secretaria_Staff (OTO Staff)
  • groups: ['Secretaria_Staff']
  • Pode criar: Trade apenas
  • Acesso: Dashboard, Inventário, Importação, Histórico, Cruzamento

Level 1: Trade User
  • groups: [] (vazio)
  • Pode criar: Nada
  • Acesso: Portal Trade apenas

🚀 COMO COMEÇAR
═══════════════════════════════════════════════════════════════════════════════

1. Instalar dependências:
   $ cd src/frontend
   $ npm install

2. Configurar ambiente (.env.local):
   VITE_API_URL=http://localhost:8000/api

3. Executar:
   $ npm run dev
   Acessa: http://localhost:5173

4. Testar:
   • Frontend redireciona para /login (sem token)
   • Faça login com usuário do backend
   • Verifique redirecionamento automático
   • Abra DevTools → Network → veja Authorization header

═══════════════════════════════════════════════════════════════════════════════
🔗 ENDPOINTS ESPERADOS DO BACKEND
═══════════════════════════════════════════════════════════════════════════════

Autenticação:
  POST   /api/auth/login/           Retorna { access, refresh, user }
  POST   /api/auth/token/refresh/   Renova access token
  GET    /api/auth/user/            Dados do usuário autenticado

Usuários:
  POST   /api/users/                Cria novo usuário
  GET    /api/users/                Lista usuários
  PATCH  /api/users/{id}/           Atualiza usuário
  DELETE /api/users/{id}/           Deleta usuário

Veja BACKEND_SETUP.md para detalhes completos.
